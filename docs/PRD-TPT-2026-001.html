§1 Overview

Problem statement, core insight, and the single measurable outcome this product must achieve.

Problem Statement

When tasks are assigned across teams, two distinct problems emerge simultaneously: (1) internal project work slips — deadlines pass silently, no one knows who is stuck; and (2) client-facing resources are invisible at the company level — management cannot see that an employee is 80% committed to a paid client before assigning them to an internal initiative.

Core Insight

→
Not all work requires the same tracking depth. Client projects need engagement visibility — who is on it and at what commitment. Internal projects need task-level tracking — what is the status, what is overdue. The system must handle both, and Admin must see the full picture across both types simultaneously.

Product Objective

Build a web application where every role gets a dashboard that answers their question in under 10 seconds — without asking anyone. Admin sees company capacity. PM sees project health. Member sees their own workload. All data is live.

Engagement Split Model

A single employee's capacity is modelled as 100%. That 100% can be split across any number of projects (client or internal). The system tracks this at project level via ProjectEngagement — separate from task-level assignment.

§6 Functional Requirements UPDATED

Key behavioral rules, including all additions from v1.1.

F-13 — Client Management

IDRequirement
F-13.1Admin and PM can create a Client with: name (required), contact person, email, phone, notes.
F-13.2A Client can have multiple associated CLIENT projects.
F-13.3Deleting a Client is blocked if active projects reference it. Archiving is allowed.
F-13.4Client list is visible to Admin; PM sees only clients whose projects they manage.

F-14 — Project Category

IDRequirement
F-14.1Every project must have a category field: CLIENT or INTERNAL. Cannot be changed after creation without Admin approval (audit-logged).
F-14.2CLIENT projects require a client_id at creation.
F-14.3The task creation UI is hidden / disabled for CLIENT projects by default. An Admin can enable it per project if ad-hoc tracking is needed.
F-14.4Project lists across the app show a CLIENT or INTERNAL badge for instant scanning.

F-15 — Project-Level Engagement

IDRequirement
F-15.1Any project (CLIENT or INTERNAL) can have employees assigned with a ProjectEngagement record containing engagement_pct.
F-15.2For CLIENT projects, ProjectEngagement is the primary tracking artefact — it replaces task-level assignment.
F-15.3For INTERNAL projects, ProjectEngagement and TaskAssignment coexist. A member can be on the project roster (ProjectEngagement) without being assigned to every task.
F-15.4The system computes each user's total active engagement = sum of engagement_pct across all their active ProjectEngagement records. This figure drives Admin dashboard.
F-15.5When assigning an employee to a project, the dialog shows their current total engagement and a preview of the new total after adding.

§8 User Stories UPDATED

Key stories covering the new client model and dashboard requirements.

Admin — Company Capacity

US-ADMIN-01 · NEW
As an Admin, I want to see every employee's engagement breakdown (client vs internal) on one screen so I can assess company capacity without asking anyone.
  • Each employee shows a stacked bar: client engagement (teal) + internal engagement (purple) = total %.
  • Employees over 100% total show an OVER badge in red; under 50% show an UNDER badge in amber.
  • I can click any employee bar to see their individual project breakdown.
  • Data reflects the current state — no manual refresh required.
MUSTNEW
US-ADMIN-02 · NEW
As an Admin, I want to see which employees are working on paid client projects vs internal projects so I can understand billable vs non-billable resource distribution.
  • Dashboard shows a summary: X employees on client work, Y on internal only, Z unassigned.
  • Client project list shows project name, client name, and the number of engaged employees.
  • I can filter the employee list by "Client only", "Internal only", "Mixed", or "Unassigned".
MUSTNEW

Client Projects

US-CLIENT-01 · NEW
As a PM, I want to create a CLIENT project linked to a client record so the system knows this engagement is for a paying client.
  • Project creation form shows a "Category" toggle: CLIENT or INTERNAL.
  • Selecting CLIENT reveals a "Client" dropdown populated from the Client list.
  • Created project appears with a CLIENT badge in all project lists.
  • Task creation is hidden on CLIENT projects by default.
MUSTNEW
US-CLIENT-02 · NEW
As a PM, I want to assign employees to a CLIENT project with an engagement percentage so I can record who is committed and at what level.
  • Assignment dialog for CLIENT projects shows "Engagement %" — no task selection.
  • The dialog shows the employee's current total engagement before and after the assignment.
  • Employees over 100% total after assignment show a warning; PM can still proceed (soft block in P1).
  • Assignment appears on Admin dashboard immediately.
MUSTNEW

Mixed Engagement

US-ENG-01 · NEW
As a team member engaged 80% on a client project, I want to also be assigned to an internal project at 20% and see my total engagement clearly on my dashboard.
  • My dashboard shows total engagement = 100%, split: 80% Client / 20% Internal.
  • The internal project's tasks appear in my task list; the client project shows only an engagement record.
  • If a PM tries to assign me an additional project, they see my current 100% total and a warning.
MUSTNEW

Core (Retained from v1.0)

US-TASK-02
As a team member, I want to update the progress percentage and status of my assigned internal task.
  • Only assignees, task creator, or a PM can update progress.
  • Progress slider: 0–100% in 5% increments.
  • Setting to 100% suggests (not forces) moving to "In Review".
  • Status change is logged with timestamp and actor name.
MUST
US-TASK-03
As a PM, I want to see all overdue tasks across my internal projects in one view.
  • A task is overdue when due date has passed and status is not Done or Archived.
  • Dashboard shows overdue count badge; clicking opens a filtered task list.
  • Overdue tasks show a red indicator in all task lists.
MUST
